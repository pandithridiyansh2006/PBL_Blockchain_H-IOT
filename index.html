Core Problem Statement

The critical vulnerability of centralized H-IoT systems to sophisticated cyberattacks‚ÄîMITM, DDoS, Data Breaches, and Replay Attacks‚Äîcompromises the confidentiality and integrity of sensitive EHRs and poses a direct, life-threatening risk to patient safety through unauthorized manipulation of medical devices. This vulnerability is exacerbated by resource constraints and the single point of failure inherent in traditional centralized architectures, necessitating a decentralized, robust, and intelligent blockchain-based security framework.

Section V + VI

Proposed Framework & Methodology

A four-layered blockchain-based security architecture that combines XChaCha20 encryption, PoA consensus, smart contract RBAC, and an ML-powered intrusion detection system.

Framework Architecture ‚Äî 4 Layers

1
IoT Data Collection Layer
Wearables, implantable medical devices (insulin pumps, pacemakers), hospital sensors (smart beds, infusion pumps). Responsible for initial patient data acquisition and secure transmission.

Encryption Layer ‚Äî XChaCha20
Data is immediately encrypted using XChaCha20, a lightweight 256-bit cipher with a 192-bit nonce. Selected for its superior speed (0.61s encryption) and suitability for low-power devices.

Blockchain Layer ‚Äî PoA Consensus + RBAC Smart Contracts
Permissioned blockchain using Proof of Authority (PoA). Trusted validator nodes (hospitals, regulatory bodies) validate transactions. Smart contracts enforce RBAC policies, manage device registration, and log all access attempts immutably.

Application & ML Layer ‚Äî Real-time Attack Detection

‚ö° XChaCha20 Encryption

Stream cipher selected for its high speed and low overhead. Uses a 256-bit key and 192-bit nonce, providing brute-force resistance. Achieves ~0.61s encryption and ~0.71s decryption on simulated H-IoT devices‚Äîwell within real-time requirements.

üîí RBAC via Smart Contracts

Every user is assigned a role (Patient, Nurse, Specialist Doctor). Smart contracts autonomously verify role-data policy matches before releasing decryption keys. Decentralized enforcement eliminates the risk of a compromised central administrator.

üß† ML-Integrated IDS

Continuous analysis of network traffic for anomalies indicative of MITM, DDoS, or Replay attacks. High-confidence detections are immediately written to the blockchain‚Äîcreating a cryptographically verifiable, tamper-proof security audit trail.
